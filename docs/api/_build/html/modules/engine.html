Engine Module

Core OCG bindings, interface, and state representation.

Bindings

CFFI bindings for edo9300 ygopro-core (OCG API v11.0).

Based on ocgapi.h and ocgapi_types.h from: https://github.com/edo9300/ygopro-core

Usage:

from src.cffi.ocg_bindings import ffi, lib

# Get version major, minor = ffi.new(“int*”), ffi.new(“int*”) lib.OCG_GetVersion(major, minor) print(f”OCG Core version: {major[0]}.{minor[0]}”)

Engine interface module for ygopro-core CFFI bindings.

Ensures library and database are properly initialized and cleaned up.

with EngineContext(auto_init_db=False) as ctx:

State

Canonical state representation for combo enumeration.

Three abstraction levels: 1. BoardSignature - For terminal board evaluation (zone-agnostic) 2. IntermediateState - For path pruning (includes OPT via legal actions) 3. ActionSpec - Standardized action representation (ygo-agent style)

Design decisions (based on analysis): - Use passcodes, not instance IDs (copies are fungible) - Zone-agnostic for board quality evaluation - Include legal actions for OPT capture - Include equip relationships (important for Fiendsmith)

Full intermediate state for path pruning.

Includes board + legal actions, which captures: - Card positions - OPT usage (via what’s still activatable) - Summon restrictions - Any other hidden state the engine tracks
